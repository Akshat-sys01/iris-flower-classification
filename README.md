# Iris Flower Classification Web Application

A full-stack Machine Learning web application that predicts the species of an Iris flower based on its physical measurements.
This project demonstrates the complete machine learning workflow — from data exploration and model training to web integration, containerization, and cloud deployment.

The application allows users to input flower measurements through a web interface and receive a prediction generated by a trained machine learning model.

## Project Overview

The Iris dataset is a classic dataset in machine learning used for classification problems.
It contains measurements of iris flowers belonging to three species:

* Setosa
* Versicolor
* Virginica

Using these measurements, a machine learning model can learn patterns that differentiate between the species.

This project builds a production-style pipeline where:

1. Data is analyzed and visualized.
2. Multiple machine learning models are trained and compared.
3. The best model is selected and saved.
4. The model is integrated into a Django web application.
5. The application is containerized using Docker.
6. The containerized application is deployed to the cloud.

---

## Machine Learning Workflow

The project follows a structured machine learning pipeline.

### 1. Data Exploration

The Iris dataset was explored using Pandas and Seaborn to understand feature relationships and class separability.

Observations:

* Petal length and petal width provide the strongest separation between classes.
* Setosa is linearly separable from the other two species.

### 2. Feature Analysis

The dataset contains four numerical features:

* Sepal Length
* Sepal Width
* Petal Length
* Petal Width

These features form the input space used for classification.

### 3. Model Training

Three classification algorithms were evaluated:

* Logistic Regression
* K-Nearest Neighbors (KNN)
* Decision Tree

Model evaluation was performed using **5-fold cross-validation**.

| Model               | Cross Validation Accuracy |
| ------------------- | ------------------------- |
| Logistic Regression | ~97%                      |
| KNN                 | ~96%                      |
| Decision Tree       | ~95%                      |

### 4. Model Selection

Logistic Regression was selected as the final model because:

* The dataset is nearly linearly separable
* The model generalizes well
* It is simple and interpretable
* It showed the most stable cross-validation performance

### 5. Model Persistence

The trained model was saved using `joblib` so it can be reused for inference inside the web application.

## Web Application Architecture

The web application is built using Django and follows the **Model-View-Template (MVT)** architecture.

User Interaction Flow:

User Input → Django View → ML Model Prediction → Response Rendered in Template

Steps:

1. User enters flower measurements in the web interface.
2. Django receives the request through a POST form submission.
3. The trained model loads and processes the input.
4. The model predicts the flower species.
5. The prediction is returned and displayed to the user.

## Running the Project Locally

### Clone Repository

```
git clone <repository-url>
cd iris-flower-classification/webapp
```

### Install Dependencies

```
pip install -r requirements.txt
```

### Run Django Server

```
python manage.py runserver
```

Open in browser:

```
http://127.0.0.1:8000
```

---

## Running with Docker

Build Docker image:

```
docker build -t iris-ml-app .
```

Run container:

```
docker run -p 8000:8000 iris-ml-app
```

Then open:

```
http://localhost:8000
```

## Deployment

The application is containerized using Docker and deployed as a **Docker web service on Render**.

Deployment workflow:

1. Push Dockerized project to GitHub.
2. Connect repository to Render.
3. Create a new Web Service with Docker environment.
4. Render automatically builds the Docker image and deploys the container.
